# Ralphthon - Hackathon Operations Website

## Tech Stack

- **Framework**: Next.js (App Router)
- **Backend**: Supabase (Auth / DB / Storage / Realtime)
- **UI**: shadcn/ui + Tailwind CSS

## Architecture Rules

1. **Participant code uses Supabase Client SDK + RLS only.**
   - NO `'use server'` directives
   - NO Server Actions
2. **Admin operations** use Route Handlers at `src/app/api/admin/*` with `service_role` key.
3. **DB schema changes** MUST go through `supabase/migrations/`.
4. **Types** come from `database.types.ts` (generated by `supabase gen types`).
5. **Client-exposed env vars**: only `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`.

## Directory Conventions

```
src/app/(participant)/   — participant-facing pages
src/app/(admin)/         — admin dashboard pages
src/app/api/admin/       — admin Route Handlers (service_role key)
src/lib/                 — shared utilities (supabase client, types, constants)
src/components/          — shared UI components
```

## Domain Rules

- **Region**: `teams.region` field (`KR` | `US`). Always filter by region where applicable.
- **Realtime**: Use Supabase Realtime (Postgres Changes) for lobster visual effect.

## Restrictions (enforced by CI)

- `'use server'` is **banned** outside `src/app/api/admin/`. CI will reject PRs that violate this.
- ESLint `no-restricted-syntax` rule also flags `'use server'` usage.
